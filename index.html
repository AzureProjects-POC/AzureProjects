Prototype Concept

Azure ChangeLens

Resource Change Explorer

Runbook + Resource Graph + Activity Logs

Problem Statement

During operational incidents, teams may investigate only the directly affected Azure resource. However, relevant changes can happen on related resources such as private endpoints, subnets, network security groups, route tables, or other shared connectivity components.

Prototype Scope

Current focus

  • Target resource: Azure Storage Account
  • Related resource discovery through Azure Resource Graph
  • Change lookup through Azure Activity Logs
  • Time options: last 1 hour, last 1 day, last 1 week, or custom window

Not in scope for this prototype

  • No root-cause claim
  • No replacement for Azure Monitor, Splunk, or Change Analysis
  • No full application/runtime dependency mapping

Technical Flow

How the Runbook Is Used

1 User enters the target resourceId.
2 User selects TimeRange: 1h, 1d, 7d, or custom.
3 Runbook discovers related Azure resources.
4 Runbook lists changes found in the selected window.
