Azure-native troubleshooting prototype

Problem Statement

During operational incidents, teams often investigate only the directly affected Azure resource. However, impactful changes may exist on operationally related resources such as private endpoints, subnets, NSGs, or route tables.

The prototype explores whether Azure-native services can surface related resource changes within a selected investigation window to improve troubleshooting context.

Example Investigation Scenario

Application connectivity issue reported for Azure Storage Account
  • Direct Storage Account review shows no obvious issue.
  • Azure ChangeLens discovers operationally related resources.
  • Recent NSG, Private Endpoint, or routing changes become visible within the selected time window.
  • Investigation teams gain faster operational context during incident analysis.

Prototype Scope

Current Prototype Scope

  • Azure Storage Account resource relationships
  • Related resource discovery using Azure Resource Graph
  • Management-plane change lookup using Activity Logs
  • Time window options: 1h, 1d, 7d, or custom

Not In Scope

  • No root-cause determination
  • No live production integration
  • No application/runtime dependency mapping
  • No replacement for Azure Monitor or SIEM tooling

Technical Flow

Runbook Workflow

1 User enters target resourceId
