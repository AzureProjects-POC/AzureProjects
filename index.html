Prototype Concept

Operational Change Context Explorer

A lightweight Azure-native exploration that surfaces recent changes across operationally related resources around an incident window.

View Sample Investigation See Architecture

Problem Statement

Incidents often start with one resource, but the relevant change may be somewhere nearby.

During operational incidents, application teams may investigate only the directly affected Azure resource while changes may have occurred in related networking, security, or connectivity resources.

Example: an on-prem application fails to access an Azure Storage Account. The storage account may show no direct change, but a related Private Endpoint, NSG, Route Table, or connectivity resource may have changed within the same time window.

Example Scenario

On-prem App
↓

↓

Current Scope & Limitations

Intentionally focused for prototype stage

What it does

Surfaces related Azure resource changes in a selected time window.

What it does not claim

It does not determine final root cause or replace observability tools.

Visibility source

Uses Azure Resource Graph and management-plane Activity Logs.

Potential Enhancements

Practical next steps

More Azure resource types

VM, App Service, Function App, Key Vault, SQL.

Cross-subscription visibility

Read-only managed identity access across participating subscriptions.

Connectivity relationships
